The Problem

Most planning tools are either too simple or too overwhelming. Google Calendar is functional but clinical and ugly. Notion has a steep learning curve. Paper planners can't be searched.

Students and young professionals end up stitching together 3-4 apps — a calendar here, tasks there, habits somewhere else — and nothing talks to each other.

The result: things fall through the cracks, planning itself becomes a chore, and people stop doing it.

/* the tool meant to reduce stress was causing it */

The answer: one calm, unified space

Competitive Analysis

Google Calendar
+ Reliable, syncs everywhere
- Clinical UI, no tasks/habits/goals, limited views
Notion
+ Powerful, flexible, great for notes
- Huge learning curve, not a real calendar, slow on mobile
Apple Calendar
+ Clean, simple, native
- Too bare, no habit/goal tracking, no task integration

None of these offer multi-calendar views (1-day, 3-day, week, month, year in one app), integrated habits, journaling, goals, and an AI assistant — all in one interface that actually feels calm.

/* the gap: something between "too bare" and "too much" */

User Personas

Maya — The Overwhelmed College Student Age 20 · Pre-med · MacBook + iPhone

Juggles 6 classes, tutoring, MCAT prep, and a social life. She's tried Notion (too complex), Google Calendar (too clinical), and paper planners (can't search them).

Needs: One place to dump everything, drag tasks onto her calendar when ready, mood tracking to catch burnout.

"I just need to get everything out of my head and into one place that doesn't stress me out more."
Jordan — The Early-Career Freelancer Age 27 · UX Designer · MacBook Pro

Manages 3-4 client projects with different deadlines. Uses Todoist + Google Calendar + Miro but hates switching apps.

Needs: Color-coded calendars per client, recurring events, notes for project ideas, quarterly goals.

"I want my planner to feel like mine, not like a project management tool my boss picked."
Alex — The Structured Student with ADHD Age 22 · CS Major · Windows + iPhone

Hard time holding tasks in working memory. If it's not written down immediately, it's gone. Needs the lowest-friction capture possible.

Needs: Instant task capture, visual time-blocking, voice input, simple 1-day view when the full week is too much.

"If I can't add it in under 5 seconds, I won't use it."

Design Process

I followed my BOLDER framework throughout this project:

Brainstorm — identified the core frustration: why do I keep abandoning planners? Because they either do too little or make me feel like I'm configuring a spaceship.

Outline — scoped the MVP: calendar with multiple views, brain dump sidebar, drag-and-drop scheduling, habits, journal, goals. Cut scope aggressively — no sharing, no notifications, no mobile app (web-first).

Layout — referenced Google Calendar's grid structure but stripped the visual noise. Established a warm, muted color palette that feels like opening a journal, not a dashboard.

Develop — built the MVP. Functionality first, aesthetics second. Got the drag-and-drop working before touching colors.

Execute — pushed it live, shared with friends and classmates for real feedback.

Refine — iterated based on user feedback (cursor states, help button requests, email verification bugs). Still looping.

Drag-and-Drop
Tasks go from dump to calendar via drag-and-drop. No modal, no form — just grab and place.

Accessibility

  • Font readability — increased base font sizes for WCAG compliance, clear type hierarchy
  • Color contrast — all text meets minimum contrast ratios against backgrounds
  • Keyboard navigation — full keyboard shortcut system for all major features
  • Dark/light mode — respects user preference, manually toggleable
  • Responsive layout — web-first with adapted mobile views (3-day to 2-day, sidebar to bottom sheet)
  • Cursor states — updated based on user feedback to clarify interactive elements

Iteration

Key changes driven by real usage and feedback:

  • Cursor feedback — testers didn't know what was clickable. Added proper cursor states across all interactive elements.
  • Email verification — verify button was redirecting to reset password. Fixed the redirect flow.
  • 3-day to 2-day on mobile — 3 columns were too cramped on phone screens. Switched to 2-day with swipe navigation.
  • Brain Dump rename — explored renaming for clarity, landed on keeping it as a recognizable concept.
  • Drag-and-drop priority reorder — originally tasks auto-sorted by priority. Added manual reorder within the same priority level.
  • Font and line weight — increased darkness across fonts and outlines after accessibility review.
Habits tracker — weekly check-off grid
